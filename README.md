# Foodora Reboot

A mobile-first meal-planning companion that tells you what to eat today, what
to prep ahead, and what to buy when — without doing the planning yourself.

## What this is

Foodora is being rebuilt as a deterministic, user-controllable food system:

- You define how many meals you want and when.
- The app builds the plan, the prep list, and the shopping list.
- Macros, calories, diet constraints, inventory, and supplement timing are
  computed deterministically — never generated by an LLM.
- A KI/AI layer (starting in M5) only formats and ranks recommendations that
  have already been validated by rule-based logic.

The case-study rationale for this architecture is documented in
[Case-Study-README.md](Case-Study-README.md).

## Stack

- **Backend & frontend framework:** Nuxt 4 with Nitro (Vue 3, TypeScript)
- **Mobile shell:** Tauri 2 (native iOS/Android targets configured in M1)
- **Database:** PostgreSQL, accessed via `postgres.js` and Drizzle ORM
- **Auth:** JWT access tokens (HS256, 15 min) + refresh tokens (7 d), `jose` +
  `bcryptjs`
- **Validation:** Zod + drizzle-zod
- **Package manager:** pnpm (managed via Corepack)
- **Tests:** Vitest (component + integration projects)

## Setup

```bash
# Use the pnpm version declared in package.jsonManager / Corepack
corepack enable
corepack prepare pnpm@10.33.0 --activate

# Install dependencies
pnpm install --frozen-lockfile

# Copy and adjust environment variables
cp .env.example .env
# Then edit .env with your local PostgreSQL credentials.

# Run database migrations
pnpm db:migrate

# Start the Nuxt dev server
pnpm dev
```

## Useful scripts

| Command | Purpose |
|---------|---------|
| `pnpm dev` | Start Nuxt dev server |
| `pnpm build` | Production build |
| `pnpm test` | Run all Vitest suites |
| `pnpm test:components` | Run component tests only |
| `pnpm db:generate` | Generate a new Drizzle migration |
| `pnpm db:migrate` | Apply pending migrations |
| `pnpm db:studio` | Open Drizzle Kit Studio |
| `pnpm tauri` | Tauri CLI wrapper |
| `pnpm build:tauri` | Static generate for Tauri |

## Project conventions

- `AGENTS.md` contains the binding rules for every contributor (branch workflow,
  secrets, CI, deployment).
- `design.md` is the authoritative design-system reference (tokens, typography,
  spacing).
- `foodora-tauri2-roadmap.md` is the milestone-spanning North Star and
  architecture reference.
- Development happens on milestone branches (`m1`, `m2`, …), never directly on
  `main`.
- Every milestone ends with a green CI run on its branch, a PR to `main`, and a
  human review before merge.
